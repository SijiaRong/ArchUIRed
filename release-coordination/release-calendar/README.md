## Overview

Each ArchUI platform releases on its own schedule, driven by feature readiness and the review timelines of its respective distribution channel (App Store, Google Play, direct web deploy). This module defines the rules that keep independent releases coherent, documents how to plan an optional coordinated release, and specifies what must be written to the shared release history so the coordination record stays current.

Details are in the submodules: release cadence rules and schema-sync timing obligations in `cadence/`, the optional coordinated-release procedure in `coordinated-release/`, the hotfix decision table and procedures in `hotfix-policy/`, and the release history file format in `release-history/`.
