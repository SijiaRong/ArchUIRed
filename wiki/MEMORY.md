## 定位

这个 wiki 层存放**不能从模块树本身推导出来的知识**：

- **为什么**这样设计，而不是"是什么"（是什么由各模块的 SPEC.md 负责）
- **跨模块的综合分析**：多个子系统共同解决了什么问题
- **架构决策记录（ADR）**：关键设计选择的动机、权衡和历史背景
- **历史上下文**：某个模块为何被废弃、重构的驱动力

## 写入规则

- **写入主体**：architect agent 或人类架构师，在做出设计决策时写入
- **写入时机**：模块被新增、重大修改或废弃时
- **不写入**：可以从代码或模块结构本身推导的事实
- **格式**：每个知识节点是一个独立的 `.md` 文件，文件名描述主题

## 与 per-module MEMORY 的关系

每个 SPEC 模块下有自己的 `memory/MEMORY.md`，存放该模块局部的、高频变化的上下文。

当一条 per-module 记忆开始影响多个 SPEC 时，应被**提升**到此处的全局 wiki 层。

```
per-SPEC memory（局部、高频）
       ↓  archui dream 提升
wiki/（全局、低频、综合）
```

## 遗忘机制

`archui dream` 命令扫描 git diff，当模块被删除或 archive 时，自动检查 wiki 里哪些页面引用了该模块，判断是否需要清理或更新。
